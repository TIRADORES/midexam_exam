Algorithm Selection & Justification

Breadth-First Search (BFS)

Explores all neighbors at the current depth before moving to the next level. Guarantees the shortest path in unweighted grids.

  • Time Complexity: O(V + E)
  • Space Complexity: O(V)
  • Optimal: Yes (for unweighted graphs)

Depth-First Search (DFS)

Explores as far as possible along each branch before backtracking. May not find the shortest path but uses less memory.

  • Time Complexity: O(V + E)
  • Space Complexity: O(V)
  • Optimal: No

Greedy Best-First Search (GBFS)

Uses a heuristic to estimate distance to goal and prioritizes nodes that seem closer. Efficient but not guaranteed to find optimal path.

  • Time Complexity: O(V log V)
  • Space Complexity: O(V)
  • Optimal: No

A* Search

Combines path cost with heuristic estimate. Guarantees shortest path with admissible heuristic. Most efficient for pathfinding.

  • Time Complexity: O(V log V)
  • Space Complexity: O(V)
  • Optimal: Yes

Why A* is Preferred for Maze Navigation

A* search is generally the best choice for maze navigation problems because it:

  • Guarantees finding the shortest path when using an admissible heuristic (like Manhattan distance)
  • Is more efficient than BFS by using a heuristic to guide the search
  • Explores fewer nodes than BFS while maintaining optimality
  • Is widely used in real-world applications and game AI

Implementation Details

Maze Generation

The maze is generated using a recursive backtracking algorithm, creating a perfect maze with exactly one solution path.

// Maze generation using recursive backtracking function generateMaze(rows, cols) { // Initialize maze with all walls let maze = Array(rows).fill().map(() => Array(cols).fill(1)); // Start from a random cell let stack = [{row: 1, col: 1}]; maze[1][1] = 0; while (stack.length > 0) { let current = stack[stack.length - 1]; let neighbors = getUnvisitedNeighbors(current, maze); if (neighbors.length > 0) { let next = neighbors[Math.floor(Math.random() * neighbors.length)]; // Remove wall between current and next let wallRow = current.row + (next.row - current.row) / 2; let wallCol = current.col + (next.col - current.col) / 2; maze[wallRow][wallCol] = 0; maze[next.row][next.col] = 0; stack.push(next); } else { stack.pop(); } } return maze; }

A* Search Implementation

The A* algorithm uses a priority queue to explore nodes with the lowest f-score (g + h), where g is the cost from start and h is the heuristic estimate to goal.

function aStar(start, goal, maze) { let openSet = new PriorityQueue((a, b) => a.f < b.f); let cameFrom = new Map(); let gScore = new Map(); let fScore = new Map(); gScore.set(key(start), 0); fScore.set(key(start), heuristic(start, goal)); openSet.enqueue({ ...start, f: fScore.get(key(start)) }); while (!openSet.isEmpty()) { let current = openSet.dequeue(); if (current.x === goal.x && current.y === goal.y) { return reconstructPath(cameFrom, current); } for (let neighbor of getNeighbors(current, maze)) { let tentativeG = gScore.get(key(current)) + 1; if (!gScore.has(key(neighbor)) || tentativeG < gScore.get(key(neighbor))) { cameFrom.set(key(neighbor), current); gScore.set(key(neighbor), tentativeG); fScore.set(key(neighbor), tentativeG + heuristic(neighbor, goal)); if (!openSet.contains(neighbor)) { openSet.enqueue({ ...neighbor, f: fScore.get(key(neighbor)) }); } } } } return null; // No path found }

Key Observations

  • BFS always finds the shortest path but visits more cells than A*
  • DFS may find a path quickly but it's rarely the shortest one
  • GBFS is fast but can get stuck in dead ends, resulting in suboptimal paths
  • A* balances efficiency and optimality, making it the best choice for most scenarios

Impact on Gameplay

The implementation demonstrates how different AI search strategies can significantly impact game behavior:

  • BFS creates methodical, predictable AI that always finds the optimal path
  • DFS can create more "explorative" AI that might discover hidden areas
  • GBFS creates goal-oriented AI that moves directly toward targets but may get stuck
  • A* creates intelligent AI that balances exploration and efficiency

These differences highlight how algorithm choice directly affects player experience and game design possibilities.
